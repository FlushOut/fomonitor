Airclic REST

Documentação de métodos

Esta é a documentação para o desenvolvedor usuário do REST Airclic.

Para acessar a documentação de erros, clique aqui.

O conteúdo abaixo explica os métodos desta API.

  1. Login

    Fará a autenticação do usuário pela aplicação. O usuário enviará o código de usuário e senha e receberá um retorno em JSON se foi positivo ou negativo.

    Parâmetros

    coduser: identificação do usuário
    password: senha de autenticação

    Retornos

    Usuário não encontrado
    { "status":false, "applications":0 }
    Usuário encontrado e com apenas uma aplicação
    { "status":true, "applications":1, "appcode":9999 }
    Usuário encontrado e com mais de uma aplicação registrada no usuário
    { "status":true, "applications":999 }

    Sendo que:
    status: valor booleano indicando se houve sucesso na realização do login
    applications: número de aplicações registradas ao usuário
    appcode: caso o applications seja 1, retorna já o código da aplicação registrada

  2. Licença de aplicativo

    Caso o usuário tenha mais de um aplicativo registrado em sua conta, é necessário que ele informe o número de licença para especificar qual aplicativo ele deseja baixar.

    Parâmetros

    coduser: identificação do usuário
    password: senha de autenticação
    licencekey: senha de autenticação

    Retornos

    Arquivo não encontrado
    { "status":false }
    Arquivo encontrado
    { "status":true }

    Sendo que:
    status: valor booleano indicando se houve sucesso na realização do login

  3. Download de aplicativo

    Tendo em posse o código da aplicação associado ao usuário, este serviço informa o caminho da aplicação no servidor.

    Parâmetros

    coduser: identificação do usuário
    password: senha de autenticação
    codapp: número de licença da aplicação

    Retornos

    Arquivo não encontrado
    { "status":false }
    Arquivo encontrado
    { "status":true, "path":abc }

    Sendo que:
    status: valor booleano indicando se houve sucesso na realização do login
    path: caminho da aplicação no servidor. A string será uma URL, apontando para o arquivo .airclic correspondente.

  4. Localização

    Registro de localização do aparelho enquanto a aplicação estiver aberta.

    Parâmetros

    coduser: identificação do usuário
    password: senha de autenticação
    lat: latitude
    lon: longitude
    speed: velocidade em metros por segundo registrada pelo aparelho
    datetime: descrição da data e hora do registro da localização no formato yyyymmddhhmmss

    Retornos

    Registrado com sucesso
    { "status":true }
    Erro no registro
    { "status":false }
